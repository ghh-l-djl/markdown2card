markdown2card它可以把 Markdown 笔记渲染成可导出的社交媒体图文卡片。

## 功能概览

- 在 Obsidian 中打开独立的 `markdown2card` 预览视图。
- 将 Markdown 转换为固定比例的图片卡片，并提供实时预览。
- 按最终渲染后的卡片高度自动分页，避免长内容被静默截断。
- 支持使用 `---` 手动强制分页。
- 将 Mermaid 图表渲染为适合卡片展示的 SVG，包含深色主题对比度修正、超大图表自动缩放，以及导出时追加原尺寸 Mermaid 图片页。
- 提供多种图片模板，包括默认、备忘录、小红书、微博、公众号、杂志、语录、终端、GitHub 和签名样式。
- 支持主题切换、封面样式、自定义字体、背景图、页脚显隐、图片裁剪缩放、表格缩放、当前页导出、全部页导出和复制到剪贴板。
- 支持预览界面语言切换，默认英文，可切换中文。
- 支持可配置导出位置。相对路径写入当前 vault，macOS/Linux 或 Windows 绝对路径写入文件系统。
- 支持 ZIP 压缩包导出，或按当前 Markdown 文件名生成 PNG 文件夹。
- 可选执行导出后动作：标记源笔记为素材、创建发布版 Markdown，并把导出资产链接写入发布版元数据。
- 集成 AI 智能重写与总结功能（基于 Gemini 3.5 Flash 等模型），可在导出后自动将笔记正文重写为小红书等平台风格的爆款营销文案。

## 模板说明

- 小红书模板保留底部互动栏，并均分点赞、收藏、评论区域。
- 微博模板使用微博风格头部，包含可上传头像、红 V 标识、当前时间、可编辑保存地址和关注按钮；该模板禁用底部页脚区域。
- 移除页脚的模板会把页脚空间释放给自动分页使用。

## AI 智能重写与营销优化

启用“导出后置操作”后，您可以开启 AI 智能重写功能，利用 Google Gemini 模型自动将笔记正文重写为适合发布平台的爆款文案：
- **安全凭证输入**：Gemini API Key 输入框采用了密码遮罩设计，防止在录屏或截屏时泄露密钥。
- **自定义模型**：支持自由输入任何您的 API Key 拥有的 Gemini 模型标识符（默认使用 `gemini-3.5-flash`）。
- **API 代理支持**：支持配置自定义的 API 请求地址/代理，以便国内用户直连使用或使用自定义的 API 中转节点。
- **提示词模板**：内置小红书爆款营销文案 Prompt 模板，支持通过 `${content}` 占位符完全自定义 AI 重写指令。
- **字数/字符数阈值**：支持设定字数阈值（默认：800字符）。若正文字数少于或等于此阈值，系统将自动跳过 AI 重写，直接导出，以避免短文本重复重写造成资源浪费。
- **社交标签提取**：自动从正文中提取话题标签（格式如 `#标签`），并将其从正文中剥离以保持正文纯净，最后将提取出来的标签以列表形式存入发布版元数据 YAML Frontmatter 的 `publish_social_tags` 字段中。
- **提示词本地化与自动同步**：AI 默认提示词模板会与用户当前的 Obsidian 插件界面语言设置（中文或英文）保持一致。切换界面语言会自动更新默认提示词模板（如已自定义修改则会予以保留），并要求 AI 模型以与输入文章一致的语言进行输出。
- **鲁棒容错机制**：如果 AI 请求失败（如网络超时或配额不足），系统会弹出错误警告，并**自动降级回退使用笔记原文**生成发布版文件，保障整个导出流程不中断。

## 开发

```bash
npm install
npm run build
```

构建会在仓库根目录生成 `main.js`，并与 `manifest.json`、`styles.css` 保持 Obsidian 社区插件预期的发布结构一致。

当前没有自动化测试脚本。UI 相关变更请运行 `npm run build`，并在 Obsidian vault 中手动验证：预览生成、自动分页、Mermaid 渲染、首次启用渲染、模板切换、主题切换、语言切换、导出路径处理、ZIP 与 PNG 文件夹导出、可选的导出后 Markdown 元数据更新、超尺寸 Mermaid 追加页导出，以及复制行为。
